# 格式快调 - 上线部署指南(腾讯云 SCF 函数 URL + 静态前端)

## 架构

```
浏览器  ──HTTPS──>  静态前端(GitHub Pages / COS)         index.html
        ──HTTPS──>  腾讯云 SCF Web 函数 + 函数 URL         /api/detect  /api/format  /api/templates
                     └ 共用 formatter.py 排版逻辑
```

- **前端** `index.html`:纯静态,托管在 GitHub Pages / 对象存储 / Webify 皆可。
- **后端** 用腾讯云 SCF **Web 函数**(Type=HTTP)+ **函数 URL**。`server.py` 的排版逻辑抽到 `formatter.py`,云函数 `cloudfunctions/format_api/index.py` 复用它,只依赖 `python-docx`。
- 文件处理在云函数 `/tmp` 完成,**排版结果以 base64 内联返回**,前端用 Blob 下载(不依赖跨实例共享存储)。

> ⚠️ 腾讯云 API 网关触发器已于 2025-06-30 完全下线,改用 SCF 原生的「Web 函数 + 函数 URL」。Webify 官方只支持静态站点 / Node SSG 框架,**没有 Python 运行时**,跑不了 `python-docx`。后端必须用 SCF 云函数。

## 第二步:部署 / 更新 SCF Web 函数

### 首次部署

```bash
set TC_SECRET_ID=AKIDxxxxxx
set TC_SECRET_KEY=xxxxxx
python deploy_scf.py
```

`deploy_scf.py` 会:
1. 删除同名旧函数(若有)
2. 创建 Web 函数(Type=HTTP, Runtime=Python3.9, Handler=`index.main_handler`, 512MB, 60s)
3. 等待 Active
4. 打印函数访问路径

> ⚠️ 首次创建后,**必须到控制台手动启用「函数 URL」**(见第三步)。SDK 3.1.100 不支持自动启用函数 URL。

### 后续更新代码

不要删除重建(会丢函数 URL 配置),用 `UpdateFunctionCode`:

```bash
python -c "
import os, base64
from pathlib import Path
from tencentcloud.common import credential
from tencentcloud.scf.v20180416 import scf_client, models
cred = credential.Credential(os.environ['TC_SECRET_ID'], os.environ['TC_SECRET_KEY'])
c = scf_client.ScfClient(cred, 'ap-shanghai')
req = models.UpdateFunctionCodeRequest()
req.FunctionName = 'format_api'
req.Namespace = 'default'
req.Handler = 'index.main_handler'
req.ZipFile = base64.b64encode(Path('dist/format_api.zip').read_bytes()).decode('ascii')
print(c.UpdateFunctionCode(req).RequestId)
"
```

## 第三步:启用函数 URL(控制台手动,一次性)

1. 打开 [SCF 控制台](https://console.cloud.tencent.com/scf/list?rid=4&ns=default) → 点 `format_api`
2. 「触发管理」标签 → 「函数 URL」区域 → 点「启用」
3. 配置:

| 字段 | 值 |
|---|---|
| 别名/版本 | `默认流量`(绑定 $LATEST,改代码立即生效) |
| 公网访问 | 启用 |
| 内网访问 | 启用 |
| CORS | 启用 |
| └ Allow-Origin | `*` |
| └ Allow-Methods | `*`(或 OPTIONS+POST+GET) |
| └ Allow-Headers | `Content-Type` |
| └ Expose-Headers | `Content-Disposition`(前端下载需要读文件名) |
| └ Allow-Credentials | 不启用(配 `*` 时必须关) |
| └ Max-Age | `86400` |
| 授权类型 | `开放`(公开服务,无需登录) |
| 参数兼容 | 启用 |

4. 启用后页面显示函数 URL,格式:`https://<appid>-<urlid>.ap-shanghai.tencentscf.com`

## 常见问题

| 现象 | 原因 / 解决 |
|---|---|
| 浏览器报 CORS 错误 | 函数 URL 的 CORS 未开启,或 Allow-Credentials 和 `*` 通配符冲突(关掉 Allow-Credentials) |
| 排版后下载报错 / 文件损坏 | 函数 URL「集成响应」未开启(默认是开的,检查一下) |
| 页面显示「后端未启动」 | `CLOUD_API_BASE` 没配、配错、或末尾多了 `/` |
| 上传大文件失败 | 函数 URL 响应体上限 6MB。超大文档请先用本地版 |
| 函数日志 `ModuleNotFoundError: lxml` | zip 里 lxml 不是 Linux wheel。用 `python build_scf.py` 重新打包(已强制 manylinux 平台) |
| 函数启动报 `SyntaxError: Non-ASCII` | `scf_bootstrap` 里用了 `python` 而不是 `python3`(SCF Python3.9 镜像里 `python` 默认指 2.7) |
| 排版结果与模板不符 | 确认 `index.html` 里 12 个模板的 `key` 与 `formatter.py` 的 `PRESET_TEMPLATES` 一致(已对齐) |
